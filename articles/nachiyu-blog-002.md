### 取得した資格
私が今回取得した資格は **JSTQB認定テスト技術者資格 Foundation Level**です。
プログラミング未経験で会社に入社して半年、QAの部署で仕事を始めて2ヶ月ほど経った頃にテストの知識をもう少し身に付けたい！と思ったことから取得を目指しました 🚩🚩

### JSTQB認定テスト技術者資格に関して

- **JSTQB認定テスト技術者資格とは**
    >ソフトウェアテスト技術振興協会（ASTER）による、ソフトウェア技術者のテスト技術を認定する資格です。本資格の運営組織であるJSTQB（Japan Software Testing Qualifications Board）は、日本のソフトウェア技術者のテスト技術を向上させることを目的とし、資格認定制度を提供しております。またJSTQBは、各国のテスト技術者認定組織が加盟しているISTQB（International Software Testing Qualifications Board）の加盟組織となっており、JSTQBが提供するテスト技術者資格は、ISTQB加盟国でも有効な資格となっております。
    出典：https://www.pearsonvue.com/jp/ja/jstqb.html　

- **Foundation Level (FL)とは**
    > 国際的に通用するソフトウェアテストの手法や技法の基礎知識を身につけたい方を対象としており、Foundation Level試験に合格することにより、システムやソフトウェアの評価・検証・テストに関する基礎的なレベルの技術を持っていることが証明されます。
    出典：https://www.pearsonvue.com/jp/ja/jstqb.html　
    

- **合格ライン**
    総出題数40問のうち正答率が6.5割を超えれば合格とされている。
   出典：https://www.qbook.jp/column/1720.html
  <br>  
- **受験スタイル**
    CBT方式(Computer Based Test式)でテストセンター受験。

## 勉強

### 受験日程と勉強期間

**受験日:** 　2026年4月12日
**勉強期間:** 2026年2月中旬 - 2026年4月頭 (約2ヶ月ほど)
**頻度:** 　　平日に30分-1時間＋移動時間などの隙間時間

JSTQB FLの試験対策を始めるにあたり、個人的に大きなポイントとなったのが「いつ受験を申し込むか」でした。今回は以下の2つの理由から、あえて最初は受験日を決めずに学習をスタートしました。

- CBT方式なので、受験日の融通がかなりきく

- 受験料が税込22,000円と、決して安くはない金額である

「先に申し込んで自分を追い込む」という方法もありますが、2万円超の受験料を無駄にはしたくありませんでした。そこで、先に学習を進め、自信をつけてから受験を申し込むことにしました。→実際に受験の申し込みを行ったのは、ある程度自信がついてきた3月後半頃でした。

CBT方式はテスト日程の選択肢が多く、直前でも空きがあれば申し込めるのが本当にありがたかったです。自分が「これならいける！」というタイミングで試験に臨めるのは、CBTならではの大きなメリットだと感じました。

### 使用教材

**使用した教材:** Udemy、公開されている公式のシラバス、テス友、Gemini


### 具体的な勉強方法

#### 1. 公式シラバスを読む (勉強開始から 1週間目-2週間目)
<br>私はまず、公式シラバスを読むところから始めてみました。
    公開されているFL用の公式シラバスがあるのですが、読み進めるのがなかなかに難しいという噂を聞いていました。どんなものかと心して読んでみたところ、**噂に違わぬ難しさ**..。
    ただ、この難しさは内容というより「日本語の読みにくさ」からくるものだなと感じました。
    もちろん出題範囲の内容を把握することはできるので、読んで損はないです。
    ですが、この「読みにくさ」を考えれば後述する「Udemyで講座を視聴する」だけでも十分かなと個人的には思いました。
    <br>もし、シラバスを読んでおきたくて読み始めたけど全然わからない！という時にはAIの力を借りることをお勧めします。わたしはAI (Gemini) に手伝ってもらいながら読んでいました 👀 
    [※シラバス解読でのAI活用方法に関しては後述します🔽](#ai活用方法)
<br>

#### 2. Udemyで講座をみる → 過去問に挑戦する（3週間目-7週間目）
<br>次にUdemyでの勉強を開始しました。Udemyを選択した理由はシンプルで、会社でプログラミングの勉強に使っていたからです。今回学習を進めるにあたり、JSTQBのコースもないかと探したところ、ありがたいことに発見できたのでそちらを使用して学習を進めました。
主に行ったことは以下の2点です。

  - **講座の視聴**
    2.5時間くらいの講座を視聴しました。出題範囲やテスト設計のことがわかりやすくまとめられていたので、出題範囲をなんとなく知ることを目的にするのであれば、前述のようにシラバスを読まずにこちらを視聴するだけでも良いかもな..と思いました。

  - **問題を解く**
   出題範囲の内容が分かってきたところで、問題に慣れるために練習問題が300問ほど用意されたコースに取り組みました。
    こちらのコースは問題に慣れるという面でとてもよかったです。
    **ただ！** 問題の模範回答でこれは違うのでは？と疑問をもったものがいくつかあったので回答が正確ではない可能性があります...

#### 3. 問題をさらに解く(隙間時間＆8週間目以降)
<br>Udemyで基礎を固めた後は、スマホアプリでの問題演習を行い、試験でどんな変化球の問題が出題されても動じない！という状態を目指しました。
その際、アプリで問題を解くだけでなく、「AIによる現在の理解度の可視化」と「苦手分野に特化した問題の作成と周回」を組み合わせて、自分の弱点を減らしていきました。

実際に使用していたアプリや、具体的な学習方法は以下の通りです。

 - 利用していたアプリ 「**テス友** 」(隙間時間におすすめ)
 → スマホアプリなので勉強したいときにすぐに取り組めてとても良かったです◎

    ::: message

   試験前の1つの指標として「テス友」のランダム出題で、どの分野でも7割以上の正答率を出せる状態を目指すと良いかもしれません。

    :::

- **AIにプロンプトを入力して利用**
    →苦手な分野を克服するために、AIのプロンプトに「現在の私の理解度リスト(分野別)」を組み込み、理解度の低いものに関して重点的に問題を作成して解くようにしました。
[※問題生成でのAI活用に関しても後述していきたいと思います 🔽](#ai活用方法)


以上の3ステップの学習を経て試験に取り組み、無事合格できました🎉
公式シラバスを読む時間を短縮できたらもう少しトータルの勉強期間も短縮できたかなと振り返って思います。


## AI活用方法

ここからは勉強していく中で私が**どのようにAIを活用したか**について詳しくまとめていきたいと思います。

#### 使用したAI : Gemini　特に「Gem」の機能

#### 活用した場面 ：主に前述した2点
- シラバスを読む時に解説をしてもらう(解説した内容を理解できたかの確認もクイズ形式で行う)
- 理解度リストを作成して、それを元に苦手な分野の問題を作成してもらう

#### なぜGemini
もともとプログラミングの勉強をしている時にGeminiのGemの機能をおすすめしていただいて利用していました。そこから今回のJSTQB FLの勉強にも活用できると思ったのでGemを使用した感じです。（今やるのであれば学習の記録がしやすいのでNotion AIを選ぶと思います..Gemもいいんですけどね..）


### Gemの使い方 (シラバス解説 / 問題作成共通)

#### 1. Gemのプロンプトを書く

私は、JSTQB FL対策に特化した学習パートナーとなるAIが欲しかったので、試行錯誤の末以下のような感じでGemのプロンプトを作成しました。（実際に使用していたものよりも若干簡略化しています）

::: message　
私が欲しかった機能
- JSTQB FLに関する質問を投げる前提を盛り込む → 前提なしにシラバスに関する質問を投げてもこちらの意図と近い形で回答してくれる
- 質問、やり取りした内容をもとにキーワード入力で確認用のクイズを生成してくれる
- 今の自分の理解度を分野別でリスト化、キーワードを入力すると、そのリストを元にJSTQB FLの形式に近い形で問題を生成してくれる

:::

```jsx
// JSTQBに関する質問をする前提
##基本設定
あなたは優秀なQA,QCエンジニアです。
QA、QC、テスト設計、JSTQB FLなどに関することを私に教えてください。
私の理解度やスキルに応じて、解説の粒度を変えてください。

//　苦手な分野から問題を出題するための理解度リスト
###　現在の私の理解度リスト（5段階の数字でレベル分けしている。5が理解度が高い。1が低い。）

#### QA,QC
- テスト設計の基礎（4）//理解度が4であることを(4)で表している
- V&V（検証と妥当性）の区別 (3)

#### JSTQB(FL)
-動的テスト(4)
-静的テスト(3)
・・・//省略

//　出題① AIに質問した内容が理解できたか確認するための問題を出題する
// 「ありがとう」を発動キーワードとして設定
### ① ユーザーから「ありがとう」と言われた場合
それまでの会話で扱っていた単元・テーマをもとに、私の理解度を問う問題を【3問】出してください。
問題構成: 基礎的な内容から2問、応用的な内容（具体的なシナリオ問題など）から1問。
出題後の流れ:
    1. ユーザーが回答したら、正解・解説を提示する。
    2. 回答結果をもとに「現在の私の理解度リスト」を更新するためのまとめや追加質問を行う。
    3. 最終的に、コピペ可能な形式で「現在の私の理解度リスト」を出力する。
    4. アップデート（変更）があった項目には【更新】や【🆙】などの印をわかりやすくつける。

//　出題② 理解度リストを元にした苦手分野の問題を出題する。
// 「苦手問題」を発動キーワードとして設定
### ② ユーザーから「苦手問題」と言われた場合
「現在の私の理解度リスト」を参照し、苦手な分野（理解度が1から3)をメインに、ランダムに【5問】テストを出してください。
問題形式: JSTQB Foundation Levelの標準形式（4肢択一式）とする。
出題後の流れ:
    1. ユーザーが回答したら、正解・解説を提示する。
    2. 回答結果をもとに「現在の私の理解度リスト」を更新するためのまとめや追加質問を行う。
    3. 最終的に、コピペ可能な形式で「現在の私の理解度リスト」を出力する。
    4. アップデートや理解度レベルの変更（上下）があった項目には【更新】や【🆙/🔽】などの印をわかりやすくつける。
判定基準: 理解度のレベルアップ判定は厳しく行ってください（例：応用問題まで完璧に根拠を持って正解できて初めて「理解」とする）。


```

<br>
#### 2. プロンプトに書いた内容を実際に使用しつつ調整する
作成したGemを実際に日々の学習で使いながら、「欲しかった機能がしっかり動いているか」を確認していきました。

- **解説機能が意図した通りに動くか**
プロンプトに「JSTQB FLに詳しいQA/QCエンジニア」という前提を組み込んだため、公式シラバスに関する大雑把な質問でも、こちらの意図を正確に汲み取って解説してくれるかを確認しました。
あわせて、学習の進捗状況を示す「理解度リスト」の更新タイミングが、自分の体感とズレていないかの調整を行い、使用感の向上を目指しました。

    **ポイント：**
    解説が少し難解に感じられたときは、プロンプトに「初学者向けに噛み砕いて解説して」といったキーワードを追加。知識がない状態からでも直感的に理解できる出力になるよう、チューニングを行ってました。


- **問題生成コマンドの検証と応用**
事前に設定した「発動キーワード（コマンド）」を入力した際に、指定したフォーマット通りに出題されるかを確認しました。

    また、基本機能の確認だけでなく、「その時の状況に応じた応用的な使い方」も模索していきました。

    **入力例：**
    「苦手問題」今回はディシジョンテーブルに関する問題を中心に10問出題してください。


最初に入力したプロンプトはシラバス解読のための「JSTQB FLに詳しいQA/QCエンジニア」という前提のみだったのですが、そこから「質問した内容の理解度を確認する問題の出題」を足したり、「理解度リストの更新」や「苦手分野の問題の出題」を加えたりしていきました。
これもっとこうしたらいいかも！と思ったらプロンプトを適宜更新して使用することでその時の自分にあわせた学習パートナーを作ることができました。かなり快適に勉強を進めることができるのでとてもおすすめです。



## さいごに
JSTQB FL取得にあたり学習した内容は実際にテスト設計/実施を行う際にとても重要な項目ばかりで、勉強している時から実務でも活かすことができました。
また、プログラミングへの知識が乏しい私でも1ヶ月半-2ヶ月あれば取得できる資格なのでウェブ開発始めたばかりだけど、なにか資格取りたい！とお考えの方にもおすすめです！

「**ちょっとずつ出題される問題の雰囲気が違う**」
というのが2ヶ月間勉強し、実際に試験を受けてみた上での感想です。

その理想に近づける方法としてAIはかなり優秀なので、ぜひ使ってみてください！今回はJSTQB FLの資格勉強にあたり使用しましたが、他の試験にも有効だと思います⭐️
